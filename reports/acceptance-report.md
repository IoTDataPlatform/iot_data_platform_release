# Отчёт о приёмке релиза

## Назначение документа

Документ фиксирует результаты проверки релизного стенда **IoT Data Platform** после развёртывания на VPS.

Цель проверки — подтвердить, что релизный стенд разворачивается, запускается, обрабатывает данные и позволяет читать накопленные данные через SQL.

---

## Проверка релизного стенда

| Проверка | Ожидаемый результат | Фактический результат | Статус |
|---|---|---|---|
| Cloud-инфраструктура создана | VPS и S3 bucket подготовлены | VPS создана, bucket создан | OK |
| Сервисные аккаунты созданы | Есть аккаунты для записи и чтения | `s3-editor` и `s3-viewer` созданы | OK |
| Репозиторий склонирован на VPS | Release-репозиторий доступен на сервере | Репозиторий склонирован | OK |
| Docker и Docker Compose установлены | `docker compose` доступен | Установка выполнена | OK |
| Git LFS установлен | Большие artifacts загружены | LFS-файлы доступны | OK |
| `.env` настроен | S3/Iceberg параметры заполнены | `.env` создан из `.env.example` | OK |
| Deployment запущен | `docker compose up -d --build` проходит успешно | Контейнеры запущены | OK |
| Source connector работает | `earth-temp-connect-source` в статусе `RUNNING` | Connector работает | OK |
| В Kafka поступают события | Topic `earth-temp` содержит сообщения | Сообщения видны в Kafbat UI | OK |
| Schema Registry работает | Subject `earth-temp-value` существует | Schema зарегистрирована | OK |
| Sink connector работает | Sink connector в статусе `RUNNING` | Connector работает | OK |
| Данные записываются в Iceberg/S3 | В S3 появляются файлы таблиц | Файлы Iceberg созданы | OK |
| Аналитик читает raw-данные | `temp_raw` доступна через SQL | `SELECT` возвращает строки | OK |
| Аналитик читает enriched-данные | `temp_enriched` доступна через SQL | `SELECT` возвращает строки | OK |
| Kafbat доступен через tunnel | UI открывается на `localhost:8080` | UI доступен через SSH tunnel | OK |

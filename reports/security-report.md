# Отчёт по безопасности релиза

## Назначение документа

Документ фиксирует результаты базовой проверки безопасности релизного стенда **IoT Data Platform**.

Основная цель проверки — убедиться, что стенд можно передавать в эксплуатацию без очевидных ошибок.

Подробная модель угроз и меры защиты описаны в `SECURITY.md`.

---

## Проверки безопасности

| Проверка | Ожидаемый результат | Фактический результат | Статус |
|---|---|---|---|
| Kafbat UI не открыт напрямую из интернета | UI не открывается по публичному IP VPS | Доступ выполняется через SSH tunnel | OK |
| Kafbat UI доступен через SSH tunnel | UI открывается на `localhost:8080` после проброса порта | UI доступен локально | OK |
| Kafka Connect REST API не открыт наружу | REST API доступен только внутри VPS / Docker network | Публичный доступ не используется | OK |
| В Git нет `.env` | В репозитории есть только `.env.example` | `.env` создаётся на VPS вручную | OK |
| Реальные S3 keys не хранятся в Git | Access key и secret key отсутствуют в репозитории | Ключи указываются только в `.env` | OK |
| Скриншоты не раскрывают секреты | На скриншотах нет Secret Access Key, паролей и токенов | Скриншоты проверены перед публикацией | OK |
| Bucket не публичный | Доступ к объектам только с авторизацией | В настройках bucket выбран доступ с авторизацией | OK |
| Для записи используется отдельный аккаунт | Запись выполняется через `s3-editor` | Аккаунт создан и используется для стенда | OK |
| Для чтения используется отдельный аккаунт | Аналитики используют `s3-viewer` | Read-only доступ вынесен отдельно | OK |
| Аналитику не выдаётся write-доступ | Аналитик может читать, но не должен изменять данные | Используется роль `storage.viewer` | OK |
| Лишние административные порты не должны быть опубликованы | Наружу не должны торчать Kafbat, Kafka Connect, внутренние сервисы | Проверяется при deployment | OK |

## Команды для ручной проверки

Проверить опубликованные контейнерные порты:

```bash
sudo docker compose ps
```

Проверить listening ports на VPS:

```bash
ss -tulpn
```

Проверить, что Kafbat UI не открывается напрямую по публичному IP:

```bash
curl http://<VPS_PUBLIC_IP>:8080
```

Ожидаемый результат:

```text
connection refused
```

или:

```text
connection timed out
```

Проверить доступ через SSH tunnel:

```bash
ssh -i ~/.ssh/<private-key> -L 8080:localhost:8080 <user>@<VPS_PUBLIC_IP>
```

После этого открыть локально:

```text
http://localhost:8080
```

Проверить отсутствие секретов в репозитории:

```bash
git grep -n "S3_SECRET_KEY\|AWS_SECRET_ACCESS_KEY\|password\|secret\|access_key"
```

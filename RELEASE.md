# План релиза: Платформа данных для IoT систем

## 1. Назначение релиза

Релиз передаёт стенд **IoT Data Platform** в состояние, пригодное для развёртывания, проверки и эксплуатации.

## 3. Пользователи системы и документация

| Пользователь | Как использует систему | Документация |
|---|---|---|
| Разработчики систем | Используют конструктор платформ и описание архитектуры для создания решений на базе платформы | основной проект IoTDataPlatform, статья на Habr |
| DevOps | Создают cloud-инфраструктуру, настраивают VPS/S3 и запускают deployment | `docs/01-cloud-vps-s3`, `docs/02-deployment` |
| SRE / эксплуатация | Проверяют Kafka, Kafka Connect, Schema Registry, consumer lag и состояние pipeline | `docs/04-monitoring` |
| Аналитики | Получают read-only доступ и выполняют SQL-запросы к накопленным данным | `docs/03-analyst-s3-read-access`, `docs/05-analyst` |
| Пользователи внешних приложений | Работают с сервисами, построенными поверх данных платформы | зависит от конкретного приложения |

---

## 4. Цели релиза

| Цель | Результат |
|---|---|
| Подготовить готовый релизный пакет | В репозитории есть deployable artifacts, инструкции и security-документы |
| Развернуть стенд на VPS | Deployment запускается через Docker Compose |
| Проверить pipeline данных | Данные проходят путь `SOURCE → STREAM → ICEBERG → S3` |
| Подготовить инструкции для ролей | Есть инструкции для DevOps, SRE и аналитиков |
| Закрыть административные интерфейсы | Kafbat доступен только через SSH tunnel |
| Зафиксировать security-проверки | Описаны угрозы, меры защиты и проверки |
| Подготовить материалы для сдачи | Есть отчёты и презентация |

## 5. Состав релиза

```text
iot_data_platform_release/
├── README.md
├── RELEASE.md
├── SECURITY.md
├── artifacts/
├── docs/
├── reports/
└── presentation/
```

`artifacts/` содержит всё, что нужно для запуска стенда:

```text
artifacts/
├── docker-compose.yml
├── .env.example
├── kafka-connect/
│   ├── Dockerfile
│   ├── connectors-pack.tar.gz
│   ├── temp-source-connector.jar
│   └── init/
└── streams/
    ├── Dockerfile
    ├── temp-enricher.jar
    ├── repo-confluent.tar.gz
    └── temp-enricher.properties
```

Большие файлы `*.jar` и `*.tar.gz` хранятся через Git LFS.
